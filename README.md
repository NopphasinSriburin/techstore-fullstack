# IT Tech Shop — เว็บ E-Commerce ขายสินค้าไอที (Full-Stack)

เว็บไซต์ขายสินค้าไอทีแบบครบวงจร พัฒนาเป็นผลงาน portfolio เพื่อแสดงทักษะ
การพัฒนา full-stack ตั้งแต่หน้าบ้าน (Vue 3) ไปจนถึงหลังบ้าน (Spring Boot + PostgreSQL)
ครอบคลุมทั้งฝั่งลูกค้า (เลือกซื้อ จ่ายเงิน ติดตามออเดอร์) และฝั่งแอดมิน
(จัดการสินค้า ออเดอร์ สต็อก และตั้งค่าร้าน) รันทั้งระบบได้ด้วย Docker คำสั่งเดียว

## ฟีเจอร์

### ฝั่งลูกค้า
- **สมัครสมาชิก / เข้าสู่ระบบ** — ระบบล็อกอินด้วย JWT
- **เลือกซื้อสินค้า** — แสดงผลแบบ responsive, ป้ายบอกสต็อก, หน้ารายละเอียดดูรูปได้หลายรูป
- **ตะกร้าสินค้า** — เพิ่ม/ลบ/ปรับจำนวน, คำนวณยอดรวมสด, ตะกร้าไม่หายเมื่อรีเฟรช (แยกตามผู้ใช้)
- **สั่งซื้อ** — ตัดสต็อกอัตโนมัติเมื่อสั่งซื้อ
- **ชำระเงิน** — ดูเลขบัญชี/QR พร้อมเพย์ของร้าน แล้วอัปโหลดสลิป
- **ประวัติออเดอร์** — ติดตามสถานะ, ยกเลิกออเดอร์ที่ยังรอชำระได้ (คืนสต็อกอัตโนมัติ)

### ฝั่งแอดมิน
- **สรุปภาพรวม** — รายได้, จำนวนออเดอร์, จำนวนสินค้า, แจ้งเตือนสินค้าหมด
- **จัดการสินค้า** — เพิ่ม/แก้/ลบ, อัปโหลดรูปหลายรูป, เลือกหมวดหมู่
- **จัดการหมวดหมู่** — สร้างหมวดหมู่ใหม่ได้ในตัวขณะเพิ่มสินค้า
- **จัดการออเดอร์** — ดูออเดอร์ทั้งหมด, ตรวจสลิปการโอน, เปลี่ยนสถานะ 5 ขั้น
- **ตั้งค่าร้าน** — กำหนดเลขบัญชี/QR พร้อมเพย์ที่แสดงให้ลูกค้า

### วงจรสถานะออเดอร์
รอชำระ (`PENDING`) → ชำระแล้ว (`PAID`) → กำลังส่ง (`SHIPPED`) → สำเร็จ (`COMPLETED`)
และยกเลิก (`CANCELLED`) ซึ่งจะคืนสต็อกอัตโนมัติ

## เทคโนโลยีที่ใช้

| ส่วน | เทคโนโลยี | เหตุผลที่เลือก |
|---|---|---|
| Frontend | **Vue 3** (Options API) | พัฒนา UI แบบแยก component, Options API เข้าใจง่ายและเป็นระเบียบ |
| HTTP | **Axios** (custom instance + interceptor) | รวมการแนบ JWT และจัดการ error 401 ไว้ที่เดียว |
| Backend | **Spring Boot 4 / Java 25** | REST API ที่แข็งแรง พิมพ์เข้มงวด มีเครื่องมือด้านความปลอดภัยครบ |
| Security | **Spring Security + JWT (jjwt)** | ระบบล็อกอินแบบ stateless ขยายระบบได้ง่าย |
| รหัสผ่าน | **BCrypt** | มาตรฐานการเข้ารหัสรหัสผ่าน |
| Database | **PostgreSQL** | ฐานข้อมูลเชิงสัมพันธ์ที่เชื่อถือได้ เหมาะกับงานออเดอร์/สต็อก |
| ORM | **Spring Data JPA / Hibernate** | ลดโค้ดซ้ำซ้อน จัดการ schema อัตโนมัติตอน dev |
| เก็บไฟล์ | **Local filesystem** (เสิร์ฟผ่าน nginx/ResourceHandler) | เก็บรูปสินค้า/สลิปแบบเรียบง่าย ไม่ต้องพึ่งบริการนอก |
| DevOps | **Docker + Docker Compose** | รันทั้งระบบด้วยคำสั่งเดียว |
| Web server | **nginx** | เสิร์ฟ frontend + proxy API ไป backend |

## โครงสร้างระบบ

```
┌─────────────────────────┐         ┌──────────────────────────────┐
│     Vue 3 Frontend      │  HTTP   │     Spring Boot Backend       │
│   (เสิร์ฟด้วย nginx)     │ ──────► │         (พอร์ต 8081)          │
│                         │  JWT    │                              │
│  • App.vue (สลับหน้า    │         │  Controllers                 │
│    ด้วย v-if)           │ ◄────── │   ├─ AuthController          │
│  • api/http.js          │  JSON   │   ├─ ProductController       │
│    (axios + interceptor)│         │   ├─ CategoryController      │
│  • components/          │         │   ├─ OrderController         │
│    ├─ shop/             │         │   └─ ShopSettingController   │
│    └─ admin/            │         │  Security                    │
│                         │         │   ├─ JwtUtil / JwtAuthFilter │
└─────────────────────────┘         │   └─ SecurityConfig          │
                                    │  Service / Repository (JPA)  │
                                    └──────────────┬───────────────┘
                                                   │
                                          ┌────────▼────────┐
                                          │   PostgreSQL    │
                                          └─────────────────┘
```

**การสลับหน้า** ตั้งใจไม่ใช้ Vue Router — `App.vue` สลับหน้าด้วย `v-if`
ทำให้โปรเจกต์เบา ไม่ต้องพึ่ง dependency เพิ่ม และเห็น logic การกันสิทธิ์ชัดเจน

**ขั้นตอนล็อกอิน:** เมื่อล็อกอินสำเร็จ server ส่ง JWT ที่มีข้อมูล username, role,
และ user id กลับมา token จะถูกเก็บฝั่ง client แล้วแนบไปกับทุก request โดย Axios
interceptor ฝั่ง server มี `JwtAuthFilter` ตรวจ token แล้ว `SecurityConfig`
อนุญาตเส้นทางตาม role

## การออกแบบด้านความปลอดภัย

ความปลอดภัยถูกออกแบบตั้งแต่ต้น ไม่ใช่ทำตอนหลัง:

- **ไม่เก็บรหัสผ่านแบบ plain text** — เข้ารหัสด้วย BCrypt ตอนสมัคร และตรวจตอนล็อกอิน
- **JWT แบบ stateless** — ไม่มี session ฝั่ง server, token พก role มาด้วย
- **กันสิทธิ์ที่ backend จริง ไม่ใช่แค่ซ่อนใน UI** — ฝั่ง Vue ซ่อนปุ่มแอดมินจากลูกค้า
  แต่ด่านจริงคือ `SecurityConfig` ที่ล็อกเส้นทาง `/api/admin/**` และการแก้สินค้า
  ให้เฉพาะ role ADMIN — ต่อให้แก้ค่าใน browser ก็เข้าไม่ได้
- **เช็คความเป็นเจ้าของ** — ลูกค้าแนบสลิป/ยกเลิกได้เฉพาะออเดอร์ของตัวเอง
  server ตรวจ username ของผู้ขอกับเจ้าของออเดอร์
- **จำกัดตามสถานะ** — ยกเลิกออเดอร์ได้เฉพาะตอน `PENDING` เท่านั้น
  ถ้าชำระแล้วต้องผ่านแอดมิน และการยกเลิกจะคืนสต็อกแบบ transaction
- **ตรวจไฟล์อัปโหลด** — รับเฉพาะไฟล์รูป, ตั้งชื่อใหม่ด้วย UUID กันชนกัน

## เริ่มต้นใช้งาน

### วิธีที่ 1: Docker (แนะนำ — ใช้คำสั่งเดียว)

ต้องมีแค่ Docker ไม่ต้องลง Java, Node, หรือ PostgreSQL เอง

```bash
git clone <ลิงก์ repo ของคุณ>
cd E-commerce
docker compose up --build
```

เปิด **http://localhost:5173** แล้วสมัครบัญชี ADMIN เพื่อเริ่มใช้งาน

### วิธีที่ 2: รันแยกแต่ละส่วน (สำหรับพัฒนา)

**ความต้องการ:** Java 25, Maven, Node.js 18+, PostgreSQL 15+

```bash
# 1. Database — สร้างฐานข้อมูล it_gadget_shop (พอร์ต 5433)

# 2. Backend
cd backend-api
# ตั้งค่า application.properties (datasource, app.jwt.secret)
mvn spring-boot:run        # รันที่ http://localhost:8081

# 3. Frontend
cd ECMShop
npm install
npm run dev                # รันที่ http://localhost:5173
```

### เริ่มใช้งานครั้งแรก
สมัครผู้ใช้ที่มี role `ADMIN` แล้วเข้าสู่ระบบเพื่อใช้หน้าแอดมิน
สร้างหมวดหมู่และสินค้าชิ้นแรก

## ปัญหาที่เจอและวิธีแก้

ปัญหาน่าสนใจที่แก้ระหว่างพัฒนา:

**1. Spring Security ชนกับ CORS filter เดิม**
การเพิ่ม Spring Security ทำให้ CORS พัง เพราะ `GlobalCorsFilter` เดิม
(ตั้ง `HIGHEST_PRECEDENCE`) กับ Security ต่างก็ใส่ header `Access-Control-Allow-Origin`
เบราว์เซอร์เลยปฏิเสธเพราะค่าซ้ำ แก้โดยลบ filter เดิม แล้วย้าย config CORS
ไปไว้รวมที่ `SecurityConfig` ที่เดียว

**2. การ migrate ข้อมูลเดิมไปคอลัมน์ NOT NULL**
การเพิ่มคอลัมน์ `status` แบบ NOT NULL บนตาราง `orders` ที่มีข้อมูลอยู่แล้ว
ทำให้ start ไม่ขึ้น เพราะแถวเก่าขัดกับ constraint แก้โดยทำเป็น migration ที่ถูกต้อง
— เพิ่มคอลัมน์แบบ nullable ก่อน เติมค่า แล้วค่อยบังคับ constraint
แทนการพึ่ง auto-DDL ของ Hibernate

**3. Lazy loading exception ตอน serialize**
รูปสินค้าตั้งเป็น LAZY ตอนแรก ทำให้ serialize ไม่ได้เมื่อดึงสินค้านอก session
แก้โดยเปลี่ยน collection รูปเป็น EAGER เพราะข้อมูลไม่เยอะ ไม่กระทบ performance

**4. ความสัมพันธ์ตารางและ field ที่บังคับ**
เพิ่มสินค้าไม่ได้เพราะ `category_id` เป็น null (Product บังคับต้องมีหมวดหมู่)
แต่ฟอร์มไม่ได้ส่งมา แก้โดยเพิ่ม CategoryController และ dropdown เลือกหมวดหมู่
ในฟอร์ม (สร้างหมวดหมู่ใหม่ได้ในตัว)
